# ConditionLoggingSystem
Multi-device telemetry collection system for Raspberry Pi

## 1. システム概要
本システムは、設置環境の気象情報、および駆動端末のハードウェア状態を包括的に記録・集約する「コンディションロギングシステム」である。

本システムは、省電力エッジ端末による計測層（Tier 1）と、高安定サーバーによる集約層（Tier 2）に分離した「2-Tier 分散アーキテクチャ」を採用し、従来課題であった端末のハングアップ（サイレント・デス）を根本から解決する。

## 2. 本システムが収集するテレメトリデータ一覧
本システムは、単なる環境測定に留まらず、システムの可観測性（Observability）を担保するため、以下の2種類のテレメトリデータを同期して収集する。

### ① 環境テレメトリ（Environment Telemetry）
- **周囲温度・湿度：** AHT25（Tier 2）および BME280（Tier 1）による多角的計測
- **大気圧：** BME280（Tier 1）による気圧変動ログ

### ② システムテレメトリ（System Metrics Telemetry）
- **バッテリー残量（%）：** PiSugarの電力マネジメント状態の監視
- **Wi-Fi電波強度（RSSI）：** 現場の通信品質およびIPv4接続安定性のログ

## 3. データ集約パイプライン
Tier 1（Zero 2 W）で生成されたテレメトリデータは、暗号化通信（rclone）による高負荷処理を完全排除し、ローカルWi-Fiを介して Tier 2（4B）の共有フォルダ（NFS）へ直送される。

192日以上の連続稼働実績を持つ Tier 2 が、これら分散されたテレメトリデータを一本化（ガッチャンコ）し、フォーマットを完全統一した上で、クラウド（Googleドライブ）への一括パブリッシュを行う。
